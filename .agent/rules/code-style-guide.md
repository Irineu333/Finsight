- A documentação é o código.
  - Evite comentários no código.
- Use padrões de programação bem estabelecidos.
- Siga o SOLID quando for a melhor abordagem.
- Prefira práticas que aumentem a manutenabilidade do código.
  - Legibilidade e simplicidade primeiro.
- Desempenho dentro dos requisitos do software. 
  - Evite otimizações prematuras.
- Boa separação de camadas. Se baseie Clean Architecture.  
- Boa separação de responsabilidade. Siga o MVVM.
- Lógica de negócio centralizada.
- Evite uso de callbacks.
